8.3. Show-hide Answer Problem

A problem with an associated answer that is revealed in the enhanced EPUB. These questions present as list items within an ordered list. If there is an answer associated with the question, the list item must have a data-answer attribute with the value of a file path to the question's answer. Otherwise do not use the data-answer attribute. There may be other list items in the ordered list that do not have the data-answer attribute applied.

Instructions

For the facilitation of show-hide answer automation in the enhanced EPUB

In order to mark up content for future show-answer automation, include the data-answer attribute with the value of the location of the answer.

Sample mark-up:

          <ol> <li data-answer="filename.xhtml#ID3236">This is a show answer question stem</li> <li>This is a question in the same ordered list without a corresponding answer</li> </ol>

where filename.xhtml#ID3236 is the URI and IDREF containing the answer text.

data-answer can only be applied to element <li>, not any other type of element, or any of the <li>'s child elements.

Special Notes

When an image is a part of a question or answer, ensure that the image is in the <li> element.

An XHTML page must not have more than 10 show-hide answer problems.

What can be marked as a show-hide answer question?

If a question is a numbered list item, it can be tagged as a show-hide answer question in the flat EPUB for the purposes of automation.

However, the following types of questions should never be tagged as show-hide answer:

  • Matching (any question with matching)
  • Multiple choice
  • Multi-select

How do I mark up the answers?

Answers must be present in the flat EPUB for the automation feature to function.

The ID of the answer must be the entire answer of the question. If there are multiple elements within an answer, all of those must be wrapped in a single element. A list item containing a nested list would be best practice.

For instance, the question 2.117 that appears earlier in the book links to the content in the screenshot below. The ID that the question refers to must encompass all of the elements in the screenshot, not just the first element, a <p> containing “2.117 (a}“.

Incorrect markup

          <p id="exe02_117" class="noindent"> <span epub:type="ordinal" class="ordinal"> <strong class="important"> <a href="kok_9781319049621_ch02_11.xhtml#ch02_exe_117">2.117</a> </strong> </span> <strong class="important">(a) </strong> </p> <table class="table compact c3 main-flow" id="ans02_untab_39">...

In this case, the only part of the answer that will be shown is the content in the p#exe02_117 element, meaning that the table and everything after the element will be excluded.

Correct markup

          <li id="exe02_117" class="noindent"> <span epub:type="ordinal" class="ordinal"> <strong class="important"> <a href="kok_9781319049621_ch02_11.xhtml#ch02_exe_117">2.117</a> </strong> </span> <strong class="important">(a)</strong> <ul> <li> <table class="table compact c3 main-flow" id="ans02_untab_39"> ... </li> ...  </ul> </li>

In this case, the li element is being linked to. Since the table and all of the other content is now nested within that li#exe02_117, and the li closing tag encompasses the table (and the figure, not shown in the code example), the entire answer will be available when the show-hide automation is performed.
