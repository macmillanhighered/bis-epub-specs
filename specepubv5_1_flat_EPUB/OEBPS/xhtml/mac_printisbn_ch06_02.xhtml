6.1. H1-H6

Create meaningful structure wherever possible by using numbered headings in a logical structure. Each section should have a numbered heading (e.g., h1) reflecting its level in the document hierarchy. For other tagged structures, specify their content with the epub:type and/or ARIA role attribute. For example, the tag that contains the preface of a book might look like <section epub:type=“preface” role="doc-preface">. Specific tags are for specific content only (i.e., the <cite> tag is only for citations. See section 4.2 Bibliographies) and should be used according to the standard. Use the most specific tag available and do not automatically wrap <div> or <span> tags around everything.

  • Numbered headings accurately reflect the document hierarchy
  • Each sectioning element has only a single heading
  • Sections and asides without headings can include an aria-label attribute to identify its structure.
  • <br /> must not be used within headings. span elements should be used to create multiline headers.
  • If an aside has a heading, it must have an aria-labelledby attribute equal to the ID of first descendant heading. Please refer to Tagging Guidelines for more information.

All asides without headings require aria-labels. These aria-labels should not be named generically but should contain meaningful information and describe the content.

For keyword asides, use the following format:

"Definition: " and then list the terms being defined. For example:
