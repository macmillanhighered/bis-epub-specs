7.1. Lists

  • Do not use lists for presentational purposes. Use them only when a list is really needed.
  • Sets of related items marked up using ordered lists ( ol) or unordered lists ( ul) elements, as appropriate. An unordered list will convey the same information regardless of the order of the items in it.
  • Glossaries, dictionaries, FAQs and similar name/value association lists should be marked up using <dl> elements.
  • Standard definition for each type of list - one tag for LOs, headings, etc. for ease of customizations.
  • When rendering as HTML list, list numbers and bullets cannot be colored, bolded, italicized, or otherwise directly styled. But custom bullets (arrows, checkmarks, etc) can be colored using Macmillan EPUB CSS with title specific color palette. For example:
  • If a list item is multiple paragraphs long, it should be tagged with “li-complex” and all of the content inside of the list must be wrapped in individual paragraph tags, not just after the first paragraph break.
  • If a list item is multiple lines long but inside one element, it should be tagged with “li-complex”.

7.1.1. List (<ol>, <ul>)

SEMANTIC DEFINITION

  • A series of items, either ordered or unordered. Lists must be declared to be ordered or unordered and can only contain list items <li>.
  • List items can contain any other element, including math, images, paragraphs, tables, and nested lists.
  • CSS is very limited in what it can do with the list bullets and numbers because the lists and bullets are dynamically generated.

ORDERED LISTS

  • In schema, for ordered lists, we allow only modifications based on the type of ordering, whether Arabic, Roman, or alphabetical.
  • Numbering can start at any number value. Numbering does not skip values.
  • Numbers cannot have colors or style (bold/italic) applied to them or any other content placed before them (parentheses, asterisks, etc).
  •               <ol> <li>Numbered list item <ol class="ol-alpha-uc"> <li>Numbered list item <ol> <li class="ol-alpha-lc">Numbered list item <ol class="ol-roman-uc"> <li>Numbered list item <ol class="ol-roman-lc"> <li>Numbered list item</li> </ol> </li> </ol> </li> </ol> </li> </ol></li> </ol>

UNORDERED LISTS

In schema, for unordered lists, we offer the default bullet, arrow, and checkmark. Due to limitations in CSS, the default bulleted list is determined by the level of list nesting: Black circle > outlined circle > black square > black square.... There is an option to start with square bullets, but the sequence will then follow as black square > outlined circle > black square > black square....

Bullets cannot have colors applied to them or any other content placed before them (parentheses, asterisks, etc).

  •               <ul> <li>Bulleted list item</li> <li>Bulleted list item</li> <li>Bulleted list item with sublist <ul> <li>Bulleted list item with sublist <ul> <li>Bulleted list item</li> </ul> </li> </ul> </li> </ul> <ul class="ul-square"> <li>Bulleted list item</li> <li>Bulleted list item</li> <li>Bulleted list item with sublist <ul> <li>Bulleted list item with sublist <ul> <li>Bulleted list item</li> </ul> </li> </ul> </li> </ul> <ul class="ul-arrow"> <li>Arrow list</li> <li>Arrow list</li> <li>Arrow list</li> </ul> <ul class="ul-checklist"> <li>Checklist</li> <li>Checklist</li> <li>Checklist</li> </ul> <ul class="ul-no-mark"> <li>No mark</li> <li>No mark</li> <li>No mark</li> </ul>

NESTED ELEMENTS WITHIN LIST ITEM

All other block-level elements may nested within list item (<li>), including paragraphs, tables, and images.

Paragraphs within list items have a space above whereas standard list items alone will not.

7.1.1.1. Multi-column lists

SINGLE LIST, MULTIPLE COLUMNS

ul/ol element must be tagged with new class list-col2, list-col3, or list-col4 (two columns, three columns, and four columns respectively). This can be used on all lists.

MULTIPLE LISTS, ONE PER COLUMN

div.group may take an additional class from col2, col3, or col4 (two columns, three columns, and four columns respectively). Group also requires, as usual, alignment classes. Please refer to 30.1 in Tagging Guidelines 3.2. div.group.col# must have a number of direct child elements equal to the # in the class. There cannot be 3 lists in a col2 div, for example.

CAUTION:

  • Aside from checkmark, arrow bullets, and bullet-less bullets, all bullets and list numbers will be positioned inside, not on a hanging indent. This is due to limitation in Vitalsource which causes list numbers and bullets to be cut off when on the outside in multi-column lists.
  • At narrower widths, lists and list items will appear one after another due to limitations in screen size. Due to the assumed pedagogical nature of the multi-column list, the content will not be moved to one fewer column, but to one single column. For example, if a reading system is narrowed to the next breakpoint width, a four column list would go to one column, not three.
  • The lists are set to break along columns so there are not widows at the top of the next column. Due to this, there may be situations where one column only has one item in it, while others have more. Therefore list items in multi-column lists are recommended to be very short, no more than a few words.
  • You must not nest lists in multi-column lists or side-by-side lists.

7.1.1.2. Complex list items

INSTRUCTIONS

For list items that require spacing around the list item due to the item’s length, please use class of li-complex as a class on the li element.

Previously this was accomplished by nesting p elements inside of li elements. p is still allowed in li but it is not necessary to achieve spacing. It is only necessary when an li has multiple paragraphs inside of it.
