4.1. Pagination

Label epub:type ARIA role Description
Pagebreak pagebreak doc-pagebreak A (sometimes valued) separator denoting the position before which a break between two contiguous pages occurs in a statically paginated media.

Page numbers are the way many people navigate within a book. For any book with a print equivalent, use the epub:type=“pagebreak” attribute to designate page numbers. Include the ISBN of the source of the page numbers in the package metadata for the book. A tag for a page number should look like <span epub:type="pagebreak" role="doc-pagebreak" id="page361" aria-label="page 361"/ > . The id must not contain spaces (e.g., page 361) or non-alphanumeric characters (e.g. page-361, page_361) separating the word page and the number.

  • Include a page-list nav to facilitate navigation.
  • Make sure that the print source is identified in a dc:source element in the package document metadata section. Additionally, include the meta refines property "source-of" with a value of "pagination".
  • Virtual Page Numbers: In digital-first titles, when there is no immediate plan to create a print counterpart and page numbers are requested, set the pagination source to the EPUB ISBN and include page numbers per usual. Page numbers should be based on XHTML document (1 XHTML doc = 1 page) and should be added mainly to accommodate reading systems that require page numbers.
  • Unless instructed otherwise by Macmillan content team, if a title has virtual page numbering, the numbering must start with roman numeral i in the frontmatter and an arabic numeral 1 in the bodymatter. The numbering should continue with arabic numerals into the backmatter.
  • Use the following standard abbreviations to number cover pages: FC (Front Cover), IFC (Inside Front Cover), IBC (Inside Back Cover), BC (Back Cover).
  • Blank pages from the print version, including blank inside covers, should not be included in the EPUB or in the nav file page list. If the pagebreaks are numerically sequential, the pagebreak element should appear directly before another pagebreak element.
  • Content that is moved from one or more pages to another is now part of the new page.
  • If the content that is moved to a new page is 3 or more pages away from its original source page, please proceed with the following annotation. Under the heading of the moved content, the original text page number should be referenced in the following manner: "(This content appears on page # of the print book)." (# represents the source page number.) This annotation should be coded as follows:
        <div class="notes" id="fri_9781319065669_BcN70qBOxO"> <p class="noindent" id="fri_9781319065669_Lr45o2GRL6"><span class="epub-only" id="fri_9781319065669_vMjR3U332N">(This content appears on page # of the print book).</span></p> </div>
  • Pagebreaks must always appear sequentially in the content. For example, page12 cannot appear before page11.
  • Pagebreak must be the first element in the element it appears in. It must appear before any text present in the element.
  • If navigable headings are moved within the EPUB, this change should be reflected in the TOC, nav, and ncx files. Those headings should appear linearly in the new order.
  • When updating pagebreaks, changes to glossary pagebreaks may be omitted.

Coding Sample 

        <section epub:type="chapter" class="chapter" id="fri_9781319065669_UelT7W185e"> <header id="fri_9781319065669_0oUqL92qBr"> <h2 epub:type="title" class="title" id="fri_9781319065669_FqzFOjioEu"> <span epub:type="pagebreak" role="doc-pagebreak" id="page1" aria-label="page 1"/> <span epub:type="label" class="label" id="fri_9781319065669_rp2o7PxSxq">Chapter </span><span epub:type="ordinal" class="ordinal" id="fri_9781319065669_7fWhbNd76i">1</span></h2> </header>

Sample coding for Continuous para: 

        <p class="noindent"><span epub:type="pagebreak" role="doc-pagebreak" id="page10" aria-label="page 10"/>Black women ... and idolatrous ... </p>
  • If a page break occurs within any floating element (including <figure> , <table> , and <aside> ), it should be moved outside of the element. For example, if a page break occurs within a <figure> tag, it should be moved outside to follow the complete element.
  • Because of the show-hide answer conversion process, elements with the data-answer attribute and elements with the epub:type answer that are being referenced by the data-answer attribute should not contain pagebreaks. The pagebreaks must be moved to the next legal pagebreak entity.
  • If a page break occurs within a <dt> element, it should be moved into the <dd> element associated with it (which should be the subsequent element).
