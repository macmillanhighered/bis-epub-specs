4.2. Navigation Documents

Label epub:type ARIA role Description
Landmarks landmarks A collection of references to well-known/recurring components within the publication
List of Illustrations loi A listing of illustrations included in the work.
List of Tables lot A listing of tables included in the work.
Table of Contents toc doc-toc A table of contents, typically appearing in the work's front matter, or at the beginning of a section.
TOC Brief toc-brief A shortened and/or specialized version of the table of contents.

The toc nav includes the full structure of the publication. Construct using ordered lists ( ol ). Each list item must contain either a single link or a heading that is followed by subsections. Use the hidden attribute to exclude subsections from visual display, if desired.

http://kb.daisy.org/publishing/docs/navigation/toc.html

  • NAV file for Navigation, which should include both a TOC and PageList. See http://kb.daisy.org/publishing/docs/navigation/toc.html and http://kb.daisy.org/publishing/docs/navigation/pagelist.html for reference.
  • The NAV file should be its own separate XHTML page. The frontmatter Table of Contents page should never be defined as the NAV file, i.e., Do not include properties="nav" in the OPF file for the TOC XHTML file, only for the actual NAV XHTML file.
  • In chapters, all end of chapter (EOC) material should be within a single XHTML file. If a book does not have its own heading for EOC then the label in the nav TOC should be “End of Chapter,” “End of Module,” “End of Section,” etc. depending on the terminology used in the book. This terminology must match.
  • End of section/chapter material must have a heading (class=”title”).
  • End of section heading can be customized by content teams in case of unconventional naming. If end of section groups are moved to the a-head level, end of section is not required. These a-head level sections must follow the conventions of a-head levels in this same section (must start a new XHTML page).

Please refer to the design memo for more guidance.

  • Any image wrapped in a <figure> tag needs to be included in the loi section of the navigation file. Below are the cases for what to use as the anchor text for the loi list items. (Note: This text should not be visible on the EPUB page.) Refer to 3.1.2 NAV Coding Sample.
    1. By default, use the figure number and title (e.g., Figure 3.1 Shifts in the Demand Curve; Infographic 2-5; Map 3.4).
    2. If there is no figure number or title, then use the caption (e.g., Chloroplasts contain highly folded thylakoid membranes.).
    3. If there is no figure number, title, or caption and the figure has a designated “unnumbered image number” in its image file name (e.g., a photo with file name 05UN11 to denote Chapter 5, 11th unnumbered image), then use that number followed by the corresponding print page number (e.g., 05UN11, page 10).
    4. If there is no figure number, title, or caption and the figure has no designated “unnumbered image number” in its file name (e.g., a page layout converted to image for EPUB), then use only the print page number(s) (e.g., pages 156-157).

4.2.2. Brief TOC

  • Brief Table of Contents. A Brief Table of Contents (not a regular TOC) represents a specialized version of the table of contents. Sometimes the heading in a Brief TOC contains additional explanatory text.
    • If a Brief TOC is present in the print book, it must be included in the EPUB as a separate XHTML file, but it must not include properties=”nav” in the OPF file for this XHTML. Therefore, the file is not treated as an EPUB Navigation document and navigation restrictions are not applicable to that file.
    • For Brief TOC's and other TOC's that are not necessary (that is, everything aside from nav.xhtml and toc.ncx), do not invent links to non-existent sections. For instance, within the Brief TOC, the links to chapters may be grouped in a manner that is not reflected in the structure of the book. So:
      • - Grouping
      • -- Chapter 1
      • -- Chapter 2

      In this example, the grouping does not have a part xhtml document that corresponds to it. Do not hyperlink this grouping to anything. This grouping should also not be represented in the nav.xhtml or toc.ncx. If the group does have a corresponding xhtml document, then it should be linked as usual.

Source: https://w3c.github.io/publ-cg/education/epub-education.html#brief-table-of-contents

4.2.3. Chapter Outlines and TOCs

  • Chapter outlines and TOCs should be tagged using proper elements. See 4.13 Navigation for sample coding. If the print book has chapter outlines, they should be tagged as brief TOCs in the EPUB. If there are no chapter outlines in the print, do not add them in the EPUB unless specially requested. Include only whatever headings and subheadings are featured in the print version.

4.2.4. Table of Contents

Provide complete navigation. Use <section> and <aside> tags in the content and the <itemref linear=“no”> tag in the manifest file to define a logical reading order. This is particularly important for academic, educational, and other complex texts.

IMPORTANT NOTES:

  • Any unique XHTML file (including bodymatter, frontmatter, backmatter, and covers) must be navigable from its own entry in the TOC. In some cases, this may mean adding content to the navigation TOC that is not already captured in the print version’s front matter table of contents.
  • Avoid linking multiple nodes/entries in the TOC (nav file) to the same location within a single XHTML page.
  • The navigation file should include at least Part, Chapter, A-head, and B-head. More specific guidance will be provided if necessary.
  • The navigation file must be titled “Navigational Table of Contents”
  • In NAV file, if the title is the main part of separate XHTML file then provide the link using XHTML filename without using ID. Example, for Part, Chapter, Ahead, FM and BM components:
          <a class="crossref" id="fri_9781319065669_7Oya3Sr8zA" href="cow_9781319098728_ch01_01.xhtml">Chapter 1</a>
  • If the title is the sub part of the XHTML then we need to use XHTML filename along with ID for linking. Example, for Bhead, Figure, table, page:
          <a class="crossref" id="fri_9781319065669_aZ0lziJl8J" href="cow_9781319098728_ch01_01.xhtml#cow_9781319098728_wBHW80j696">Bhead title</a>
  • When an XHTML file is being referenced, in the navigation or anywhere else in the book, never refer to the body element's id attribute in the link. Please refer to a descendant of the body element, such as a section, header, or h# element.
  • Include headings in the navigation document for items like list of figures, list of tables, page list, and others. For example:
           <nav id="fri_9781319065669_B28MpwXpTh” epub:type="page-list” role=“doc-pagelist” class="hidden”> <h2 id="fri_9781319065669_B29MpwXpTh”>Page List</h2> <ol id="fri_9781319065669_B30MpwXpTh”> <li id="fri_9781319065669_B31MpwXpTh”><a id="fri_9781319065669_B32MpwXpTh” class="crossref" href="rub_9781464182495_cover.xhtml#pageFC">FC</a></li>
  • Order of XHTML files in navigation (NAV doc) must match the spine in the OPF. This is now checked starting in EPUBCheck 4.2.0.
  • Order of page numbers in the EPUB must match the order of page break links in the page list of navigation (NAV doc). This is now checked starting in EPUBCheck 4.2.0.

4.2.5.1. Landmarks

Landmarks are required for navigating the major sections of an EPUB. epub:type should be placed on the a element.

This list of landmarks is not exhaustive. Additional links to front and back matter elements should be included as well.

epub:type text
cover Cover
bodymatter Begin Reading
toc Table of Contents
bibliography Bibliography
index Index
glossary Glossary

If used, attributes aria-label and title attributes must be unique throughout landmarks. Numbering may be used to make the aria-labels or titles unique.
