7.3.1. Image

  • Text alternatives must be provided for all significant images using the alt attribute
  • Descriptions provided for all significant images
  • Leave the alt attribute empty for all decorative images that contain no content.
  • All images used in the package must be in an OEBPS subfolder called “images”. The EPUB package must not contain any unused images. Images without any callout in XHTML should be removed from the “images” folder and from the OPF file.
  • Please refer to our separate "Image Prep for Digital Delivery" document for detailed image requirements.
  • Image Resolution: All images in the EPUB package must be 150 dpi.
  • Image Format: All images should be in the format of PNG, except the cover image file which should be in JPEG format.
  • Image Size: All images should be below 4 million pixels.
  • Image Crediting: Credit information should be captured in the embedded image file only and must be removed from the surrounding text to avoid duplication of content.
  • Composite Images: When images are made up of multiple parts (a, b, c), the separate image files need to be merged into one image. The callout within text would link to the combined version. If Macmillan Learning doesn’t provide the merged version of the image, the vendor is expected to perform that work. Although Macmillan Learning caption copy is typically written using the a, b, c, references, if you encounter copy that references (left, right, top, bottom), the copy may need edits. Query the Macmillan Learning project manager requesting the EPUB to resolve the issues.
  • Do not use more than one <img> inside a single <figure>. If a set of images is grouped together and shares a caption and/or credit, it should be merged into a single composite image file as described above.
  • Images that span spreads must be captured as a single image.
  • Use the sRGB color space. The CMYK and GRAY image profiles should not be used as they will result in image validation errors.
  • Use PNG format for all images, including:
    • line art/vector images
    • fallback images for math equations
    • bitmap images containing text
  • Use PNG format with type-optimized anti-aliasing. Note: for any image with <figure> element and class=”lm_img_lightbox” attribute, the included PNG file should not have a transparent background, because it will not display correctly in the pop-up window in the enhanced EPUB version.
  • Images should not exceed 2000 pixels in width.
  • Create dingbats and icons as transparent PNG files with no white background.
  • Any ebook with 10 images or more should have some image compression software run on the images folder. This is to reduce EPUB file size and thus make it easier and faster to load on a variety of devices.

7.3.2. Icons in headings

When an icon image is required to float beside a heading, place the icon after the header element and apply the classes "decorative" and "logo". The icon image must not be placed inside of the header element, as this interferes with CFI generation. The Universal SCSS has been updated accordingly.

7.3.3. Decorative Image Treatment

Any image that is set as in-line (icons for example) should NOT use the <figure> tag. Any image that is considered decorative must be tagged with role set to “presentation”  and alt text set to null.

          <img class="decorative size-med left" id="fri_9781319065669_4ErYqI5gOB" src="imagename.png" role="presentation" aria-hidden=”true” alt=""/>

Decorative images must also be tagged with the label aria-hidden="true".

See 6.13 Use Image Descriptions and Alt Text for further details.

7.3.4. QR Code Treatment

If part or whole image includes a QR code, the following will need to be adjusted to remove and replace instance of QR code:

  • If QR occurs by itself in an image, please omit image from EPUB and replace with link that was provided by QR code as href in EPUB
  • If QR code is part of composite image please update image to delete QR code and update alt text to remove mention.  Include link that was provided by QR code as href in EPUB.
