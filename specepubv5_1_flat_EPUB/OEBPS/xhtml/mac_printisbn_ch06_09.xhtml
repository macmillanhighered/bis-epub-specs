6.7. Math

MathML makes mathematical equations accessible to everyone by eliminating the ambiguity of a verbal description of a picture. There are many tools available to support MathML creation.

  • MathML markup must be included as the primary display for all math content.
  • role="math" must never be used on the container of any MathML element (such as span.math or div.informalequation).
  • aria-labels must never be used to present alt-text for the math container element. The alttext attribute on element <math> must be used to represent the alt-text for the math content.
  • The math element itself requires the MathML XML namespace, without a prefix. So: <math xmlns="http://www.w3.org/1998/Math/MathML" alttext="alttext"> This namespace must never be applied to the HTML element.
  • The fallback image which is inside the annotation-xml element must have the XHTML namespace applied to it. So:
    • <annotation-xml name="alternate-representation"
    • encoding="application/xhtml+xml">
    • <img xmlns="http://www.w3.org/1999/xhtml"
    • src="../images/MATH/fee_9781319218508_ch01_02EQ_0001.png"
    • alt="cap y equals cap c plus cap i plus cap g plus EX minus IM" />
    • </annotation-xml>
  • Punctuation that is not part of math content should be set within the div or span wrapper that the math is in ( span.math , div.informalequation ) rather than as MathML itself. This is to prevent the punctuation breaking to the next line and also to have consistent rendering for punctuation in math content.
        <p class="indent" id="fra_9781319188016_9hogbobyfG">The statistic is <span class="math" id="fra_9781319188016_tlWNyBtZy0" tabindex="0"> <math display="inline" id="fra_9781319188016_xFFYjxGktH" xmlns="http://www.w3.org/1998/Math/MathML" alttext="ModifyingAbove p With caret"><semantics id="fra_9781319188016_RQgWWvs3Ba"><mover id="fra_9781319188016_b3H71L0Mh1" accent='true'><mi id="fra_9781319188016_oGoY2xKdMP">p</mi><mo id="fra_9781319188016_HtSdHnpqYV">&#x005E;</mo></mover><annotation-xml id="fra_9781319188016_CBVX2VdJDR" name="alternate-representation" encoding="application/xhtml+xml" ><img id="fra_9781319188016_7SvPhbPiWh" src="../images/EQp1-006.png" alt="ModifyingAbove p With caret"/></annotation-xml></semantics></math>,</span> the proportion of the sample who say they believe in ghosts.</p>
  • Math content cannot depend on a specific font. If a specific character is required and it is not available in a standard font, it must be replaced by an image. Users have the ability to override publisher fonts, so it must always be assumed that they will. For math, please refer to this appendix to view math characters that cannot be rendered in MathML.
  • Additionally, please use MathJax 2.7, which has a11y features that automatically generate language for math formulas. Comp should create fallback images for any math displayed with MathML and use the MathJax autogenerated language as aria-label and alt text for the fallback images, which should be in PNG format only. MathSpeak rules should be used when autogenerating the text via MathJax.
  • For MathML content that does not use MathJax 2.7 and therefore cannot have descriptive language autogenerated, the comp should query Macmillan Learning to ask if we want fallback images with alt text/aria-labels specially written in that case.
  • <div class="informalequation" id="fri_9781319065669_mJazrQKqrk" tabindex="0"> <math xmlns="http://www.w3.org/1998/Math/MathML" id="fri_9781319065669_dKexQUZpIm" alttext="[spoken equivalent]"> ... <math> </div>
  • Important: Do not include the attribute tabindex on MathML that is inside of an <a> element. This confuses accessibility technology.
  • If MathML element <annotation-xml> contains HTML content, it must have the following attribute and value:
    • encoding="application/xhtml+xml"
  • Unless the content falls under other criteria for MathML rendering, do not render a number followed by a unit of measurement as MathML. For example, “10,000 km” does not need to be rendered as MathML on its own. However, if it were part of a larger mathematical or other scientific expression that did need to be captured as MathML, then the unit of measurement should be captured as MathML as well.
  • Standalone equations should appear after glossary keyterms and before figures called out in paragraphs.

6.7.1. In-Line vs. Display/Offset Fractions

In print, math that occurs "inline" (within a paragraph of text) varies in styling vs. equations that are on their own line or surrounded by minimal text, referred to as “display” equations. For example, in a display equation the fractions are sized so that the font size of numerator and denominator match the base font; in an inline equation the font size of numerator and denominator is smaller.
