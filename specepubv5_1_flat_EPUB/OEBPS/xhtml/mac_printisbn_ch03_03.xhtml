3.2. Conformance

Meet WCAG 2.0 Level AA requirements. To meet the requirements for a “set of Web pages:” overall reading order of content from page to page must be logical; all content documents must have a logical title; etc. All parts of all pages must be compliant. Conforming alternate versions of the EPUB are not acceptable substitutes for accessibility of the content.

https://www.w3.org/Submission/epub-a11y

3.2.1. Requirements Beyond WCAG 2.0 AA

Meet unique requirements of EPUB, which go beyond WCAG 2.0. Provide page markers and a means of accessing them for any work that also exists in a statically paginated version.

https://www.w3.org/Submission/epub-a11y/#sec-epub-req

3.2.2. Conformance Reporting

To indicate that an EPUB Publication conforms to the accessibility requirements of this specification, it must include a conformsTo property [DCTERMS] and an a11y:certifiedBy property [Accessibility Vocab].

Include a <conformsTo>  property that specifies the IRI of the dated version of this specification and include one of the following fragment identifiers that identifies the conformance level achieved: i.e., wcag-aa. Include an <a11y:certifiedBy> property that specifies the name of the party that certified the content.

For ALL Macmillan titles, insert the following information into the EPUB3 file. 

          <link rel="dcterms:conformsTo" href="http://www.idpf.org/epub/a11y/accessibility-20170105.html#wcag-aa"/>

https://www.w3.org/Submission/epub-a11y/#sec-conf-reporting

For ALL copyright 2019 titles and beyond, insert the following information in the OPF of the EPUB3 file: 

          <meta property=”a11y:certifiedBy”>Benetech</meta> <meta property="a11y:certifierCredential">https://bornaccessible.org/certification/gca-credential/</meta>
