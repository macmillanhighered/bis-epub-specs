9.3. For titles not using Macmillan EPUB CSS

  • The CSS file should be as brief as possible.
    • Do not use redundancies.
    • Do not keep elements in the CSS that are not actually in the book.

9.3.1. Text Styling for Titles Not Using Standard CSS

  • Never use drop caps, stick caps, or other similar decorative font styling.
  • Do not set text to black in CSS; only use spot color. The default color for any text in any reading system or device is black and background is white. Overriding this default property through CSS will introduce legibility issues. Do not use the below CSS properties
    • color: #000000;
    • color: #ffffff;
    • background-color: #000000;
    • background-color: #ffffff;
  • White text cannot be used. When converting a print design that has white text on a colored background, use the following standards alternatives. In either case, it is required to verify the contrast of the updated colors for accessibility.
    • Default: Replace white text with #f3f3f3; change the background color to the closest color from the list of approved options:
      • #a51426    dark red
      • #dd5714    orange
      • #a83d0e    dark orange
      • #74600c    dark yellow
      • #2c5e20    dark green
      • #16664b    dark teal
      • #205460    dark aqua
      • #006eb8    blue
      • #004c72    dark blue
      • #865f7f    purple
      • #543b50    dark purple
      • #882345    magenta
      • #562135    dark magenta
      • #686868    dark gray
      • #383838    very dark gray
    • If the editorial reviewer does not approve the default treatment, the only acceptable alternative is to remove the background color and change the text color to either one of the above colors or one of the following:
      • #da1b2c    Macmillan red
      • #b79e25    yellow
      • #3b822e    green
      • #218466    teal
  • Default highlight/selection color should be: #dddddd

https://css-tricks.com/overriding-the-default-text-selection-color-with-css/

  • Do not manipulate styling of XHTML element <a> properties using CSS , e.g.: a { text-decoration: none; color: #008000; }
  • Do not use <b> for bolding of glossterms in the glossary or lists of keywords; only use <b> for semantic emphasis of a keyword appearing within the narrative text (<dfn epub:type=”keyword”>). Glossary styling should be managed via CSS.
  • Do not use in-line CSS styling, e.g. This cookbook is hot! This cookbook is <span class="red">hot!</span> stylesheet.css span.red { color:red; }

    NOT: This cookbook is <span style="color:red;">hot!</span>

9.3.3. Text Alignment for Titles Not Using Standard CSS

  • Unless otherwise instructed, text in the EPUB should be set Flush Left, not Justified.
  • Do not use non-breaking spaces ( &#x00A0; or &#160; ) for aligning content.

9.3.4. Restrictions on Class Values

Never use the following values for naming classes:

  • "gadget"
  • "assessment"
  • "dataset"
  • "multiplevideo"
  • "widget"
  • "exwidget"
  • "ext-gadget"
  • "brightcovevideo"
  • "footnote"

Use the following class values only as specified:

  • “keyword”  should be used only in <a>  or <dfn>  in bodymatter; it should never be used in the glossary file.
  • "lm_img_lightbox” should be used only in <figure>
