3.3. Best Practices

3.3.1. Separate Presentation and Content

  • Visual reading is only one way of accessing content. Do not use visual-only cues such as colored text, font size or positioning as the only clue to the meaning or importance of a word or section.
  • Avoid using tables or pictures of text to control the appearance of the content. As a last resort, page layouts should only be captured as images in cases where elements such as marginal annotations or highlighted/marked up content cannot be rendered properly in HTML and still maintain pedagogical meaning, e.g. marked up student essays or bibliographies in a Composition title or complex infographics. In the example of essays, bibliographies, or other such content spanning more than one page, ensure images are sized and cropped with equal pixel width and spacing for visual consistency. Note: the entire essay, passage, bibliography, etc. must be taken as images, even sections without any mark-up, to ensure visual consistency with the marked up sections. All such images must use long alt text to capture the full text of the essay as well as to describe any markup.
    • Incorrect Example: The first citation (“Heims, Marjorie…”) should be treated as an image so that it looks consistent in font and alignment with the marked up citation below it:
    • Correct Example: The “Works Cited” section of this essay is unannotated, but it has been captured as an image to appear consistent in font and alignment with the annotated portion above it:
  • The meaning of the content should be the same both with and without any CSS styles or formatting applied. Use the epub:type and/or the ARIA role attribute with the relevant structural tag to establish reading order, and meaningful styling, such as sidebars and alerts.

Refer to: http://kb.daisy.org/publishing/docs/html/separation.html

3.3.2. Use Images Only for Pictures, Not for Tables or Text

Any content embedded in an image is not available to visually impaired readers. If the textual contents of a table or image are required for comprehension of the document, use proper and complete markup for text and tabular data, including headers and scope attributes for tables. Include a caption to give context to the table. For complex tables, include a summary of the structure to guide the reader. (See 3.6 Tables.) Footnotes should appear with the table--wrap the table and footnotes in a figure tag.

http://kb.daisy.org/publishing/docs/html/tables.html

3.3.3. Use Image Descriptions and Alt Text

Every image should have a description, caption, or alt text unless it is solely decorative (contains no content).

All images must have an alt attribute.

The alt text should be meaningful and a reasonable substitution for the image itself. It must not merely indicate that the image is described by the surrounding text.

When an image is marked as purely decorative and has an empty alt attribute, it is removed from the accessibility tree and thus cannot be accessed by the AT user who will not know that there is an image present to be described by the surrounding text.

If an image is purely decorative and has no meaningful impact on the surrounding content, it does not need alt-text and its alt attribute must be empty/null, e.g. alt="" and have the attribute aria-hidden with value true.

It must reference an id attribute of another element that contains the description. This means, the value of the aria-describedby attribute must be the same as the value of the id attribute for the description of the image:

          <img class="infographic" id="fri_9781319065669_bP38cjLJYN" src="imageofsomething.png" alt="short description" aria-describedby="fri_9781319065669_ch26_01_Ht2l1hX3Ky"/> <p class="noindent" id="fri_9781319065669_Ht2l1hX3Ky">Image Description for imageofsomething.png</p>

If they don't match, the sighted reader will not know the relationship between the image and the paragraph that carries the description of that image.

  • Alternative text should not be redundant (e.g. be the same as caption or body text).
  • Alternative text should not use the phrases "image of…" or "graphic of…".
  • Alternative text should not exceed 100 characters; otherwise it must be shortened or move to longer descriptions. Long descriptions need to use the aria-describedby attribute.

http://kb.daisy.org/publishing/docs/html/images.html

An icon may require alt-text if it is informative and not merely decorative. This is especially necessary if no other text in the same line of text next to the icon conveys the same meaning or if the icon recurs throughout the book.

Best practices documentation for icon alt-text presents scenarios where alt-text is and is not necessary.

Icons that have their meanings described in the beginning of the book should have alt-text created for them. These are typically dataset, widget, video, and caution icons.

For example, if the word "caution" is not used when a caution icon appears, then the non-sighted user would not be aware of the full semantic context of the paragraph.

Publisher logo also requires alt-text. This will require internal ML definition for alt-text for each publisher logo that we use. Alt-text for Macmillan company logo is alt="Logo: Macmillan Learning".

3.3.4. Define the Language(s)

To make sure each word will be rendered correctly, specify the default language of the content in the root html tag. Indicate any words, phrases or passages in a different language by using the xml:lang attribute: <span xml:lang=“fr” lang=“fr” id="fri_9781319065669_eB73iWACec">rue Saint-Andre-des-Arts</span>.

  • The lang and xml:lang attributes are set on all root html elements
  • The lang and xml:lang attributes are set on all instances of foreign language prose

http://kb.daisy.org/publishing/docs/html/lang.html

3.3.6. Semantics

Include semantic information to describe the content of a tag (examples: foreword, chapter, glossary, index). A section tag for the table of contents would look like <section epub:type=“toc” role="doc-toc"> or a list of definitions in a glossary would be tagged with <dl epub:type=“glossary”>. Refer to the Semantic Structure section of this document for a list of vocabulary to be used within Macmillan Learning titles.

  • epub:type: http://kb.daisy.org/publishing/docs/html/epub-type.html Refer to the EPUB 3 Structural Semantics Vocabulary as defined at http://idpf.org/epub/vocab/structure/ for further details and guidance.
  • ARIA role: http://kb.daisy.org/publishing/docs/html/roles.html Refer to the ARIA mapping guide (https://idpf.github.io/epub-guides/aria-mapping/) to include ARIA role counterparts to epub:types.

http://kb.daisy.org/publishing/docs/html/sections.html

3.3.6.1. The epub:type Attribute

Logical reading order is crucial for users of assistive technologies. Use the epub:type attribute along with any of the terms defined in the EPUB Structural Semantics Vocabulary to identify structural sections and significant content structures, such as dedication pages, acknowledgements, epilogues, etc.

http://kb.daisy.org/publishing/docs/html/epub-type.html

3.3.6.2. ARIA

epub:type attributes for content structures should be paired with ARIA role counterparts. This improves performance with assistive technologies.

https://www.w3.org/TR/html-aria/

3.3.7. Entities

Use appropriate entities for characters. For instance, for ellipsis character &#8230; or the ellipsis character (…) should be used rather than … (three periods).
