Whenever the print book has key term definitions in the margins, the term should be moved in line directly following the paragraph or list where the term is first cited, as this example illustrates. When there is more than one term in a paragraph or a list, stack them in the order they fall.

NOTE: The term and definition following a paragraph or list is the default standard. If the Macmillan reviewer specifically requests this treatment not be followed, the only acceptable alternatives are to move the terms to the end of the chapter or a specified section or to remove them entirely. In any of these cases, the glossary linking instructions below must be followed properly.

In some titles, the print book may include a bolded key term in the margin without any definition in order to visually point the reader to a paragraph that contains a key term. This repeated term is extraneous in the EPUB and should always be removed. PDF example:

Glossary linking: The anchor tag for the key term in the paragraph should always link to the term in glossary page, i.e., the IDREF should refer to the ID in <dt> in the glossary XHTML page.

Glossary links may link back to the first instance of the glossary term. It is not required to do so, but may be done upon request.

Important Note: The order of elements is required to follow the coding sample below (i.e., <a href> must be placed inside <dfn>).

In Macmillan EPUB CSS, bolding is applied directly to the <dfn> element, so nesting a <b> element within is unnecessary. Therefore the default style for <dfn> will be bold, normal, but this may be customized by the compositor upon request within the CSS and customizations should not be done with nested inline styles. Nested inline styles may be used for unique cases.

Linking back from glossary: Remove page-break anchor references within the glossary itself. Users can navigate back to the main text from the glossary by using the back button in their reading system.

Back-linking in High School Titles: All glossary definitions should have backlinks to the first instance of a keyword being mentioned. Note that this is supplemental to the above rule regarding using page-break anchor references. The linking must be back to the keyword, not the page-break anchor. Follow the format for footnote back-linking.

Multi-paragraph definitions: When a definition (HTML element dd) is multiple paragraphs long, each paragraph must be tagged as a p element. In Macmillan EPUB CSS, the first paragraph should be p.noindent and all subsequent paragraphs should be p.indent. In other titles, the first paragraph should not be indented and the rest of the paragraphs should be indented.

5.4.1.2. Packaging of Glossary Elements

  • When the first occurrence of glossary key terms is highlighted throughout the text, the vendor should insert the Key term and Definition directly following the paragraph where cited, per the coding example provided.
  • If inside of the main glossary section, element dfn must not have an epub:type with value of keyword.
  • Marginal keywords should only be followed for definitions appearing in the margins. If the book contains glossary terms or definitions at the end of each chapter or at the end of the book, these sections are to be maintained and coded following the examples we provide. They should not also be copied and inserted into the basal text.
  • If a book contains end of chapter glossary definitions (not just terms), these should be collected at the back of the book for the main glossary.
  • By special request, these may also remain in the end of chapter material per the aside tagging guidelines for keyword definitions outlined in the section below “Keyword definitions throughout”; however, these definitions must appear in the end of the book as well in the main glossary. This is to enable pop-up glossary in the enhanced EPUB.
  • For books that have more than one glossary (e.g., one English and one Spanish language version), each one should be tagged with   epub:type=”glossary”, but only the primary glossary file should have the additional attribute class=”glossary”. If it is unclear which is the primary glossary, query the ML project manager requesting the EPUB.
  • Glossary terms do not require a colon after them since they are placed by default on another line.

5.4.1.4. Bilingual Glossaries

In high school titles, we are required to have bilingual glossaries. These bilingual glossaries must be treated as tables for ease of use.

However, due to requirements for the automation of pop-up glossaries, the tabular bilingual glossaries present a unique challenge:

In the enhanced EPUB, the pop-up glossary requires a single glossary in the back of the book that the linked key terms must point to (which facilitates the automatic enhancement).

In the flat EPUB, the key terms must point to the English term in the bilingual glossary, which is presented beside the foreign language term in a table

The key terms throughout the flat EPUB must link to the tabular bilingual glossary

A separate English-language glossary document shall be created outside of the EPUB package.

This document shall be delivered to media production for use in facilitating the pop-up glossary automation and be incorporated into the EPUB package.

5.4.1.5.1. Glossary elements (<dl>, <dt>, <dd>, <dfn>)

SEMANTIC DEFINITION

The description and definition elements, in our schema, serve as the glossary elements. Glossaries in our EPUBs appear in two different ways: throughout the book as definitions and at the back of the book grouped together. However, both types share the common description list elements described below:

  • Description list <dl>

    A grouping of term-description pairs, typically used in a glossary.

  • Description term <dt>

    A term to be defined. In our schema, it must be followed by a <dd> element (description). In HTML in general, multiple <dt> elements (multiple terms) can be defined by a single <dd> description. However, since we use our description lists exclusively for glossaries, each description must correspond to one term.

  • Description details <dd>

    A description of a term. It can contain other elements, such as multiple paragraphs. In this way it is more flexible than <dt>

  • Definition <dfn>

    A defined term used in the context of the running content.

5.4.1.5.2. Back of the book glossary

The glossary in the back of the book, or section-glossary, has strict rules. It must follow the following hierarchy:

  • section.section-glossary
    • dl.glossary Note: There should only be one dl.glossary with dt + dd pairs occurring one after another.
      • dt.glossterm
        • dfn.keyword must never appear in section.section-glossary
      • dd.glossdef
        • optional: any HTML elements, such as paragraphs. Note: If multiple elements are used, all of the content within the dd element must be wrapped in HTML tags. That is, none of the text should be left untagged in the dd element.
5.4.1.5.3. Keyword definitions throughout

Definitions may also appear throughout the book. They should be structured as such:

  • aside.glossary There should be one aside and dl per definition. This aside element must have an aria-label with the value "Definition: " followed by the text within the child dt elements.
    • dl.glossary
      • dt.glossterm
        • dfn.keyword We do not recommend this being put in. dfn should only really be used to indicate a keyword appearing in the actual text, and that should contain a link to the definition in the back of the book glossary.
      • dd.glossdef
        • optional: any HTML elements, such as paragraphs. Note: If multiple elements are used, all of the content within the dd element must be wrapped in HTML tags. That is, none of the text should be left untagged in the dd element.

With our styling, the key word asides render as light gray text. The dfn element will contain a link to the relevant glossary item.
