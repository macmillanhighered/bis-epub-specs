8.1. Show-Hint Problem

For the facilitation of show-hint automation in the enhanced EPUB.

In order to mark up content for future show-hint automation, include the data-hint attribute with the value of the location of the hint.

        <ol> <li data-answer="filename.xhtml#ID3236" data-hint="filename.xhtml#ID3237">This is a show answer question stem containing a data-hint</li> <li>This is a question in the same ordered list without a corresponding answer</li> </ol>

where filename.xhtml#ID3237 is the URI and IDREF containing the hint text.

data-hint can only be applied to element <li>, not any other type of element, or any of the <li>'s child elements.

Special Notes

When an image is a part of a question or hint, ensure that the image is child of the <li> element.

What can be marked as a show-hint question?

If a question is a numbered list item, it can be tagged as a show-hint question in the flat EPUB for the purposes of automation.

However, the following types of questions should never be tagged as show-hint:

  • Matching (any question with matching)
  • Multiple choice
  • Multi-select

How do I mark up the hints?

Hints must be present in the flat EPUB for the automation feature to function.

The ID of the answer must be the entire answer of the question. If there are multiple elements within an answer, all of those must be wrapped in a single element. A list item containing a nested list would be best practice.

Incorrect markup

          <p id="exe02_117" class="noindent"> <span epub:type="ordinal" class="ordinal"> <strong class="important"> <a href="kok_9781319049621_ch02_11.xhtml#ch02_exe_117">2.117</a> </strong> </span> <strong class="important">(a) </strong> </p> <table class="table compact c3 main-flow" id="ans02_untab_39">...

In this case, the only part of the hint that will be shown is the content in the p#exe02_117 element, meaning that the table and everything after the element will be excluded.

Correct markup

          <li id="exe02_117" class="noindent"> <span epub:type="ordinal" class="ordinal"> <strong class="important"> <a href="kok_9781319049621_ch02_11.xhtml#ch02_exe_117">2.117</a> </strong> </span> <strong class="important">(a)</strong> <ul> <li> <table class="table compact c3 main-flow" id="ans02_untab_39"> ... </li> ... </ul> </li>

In this case, the li element is being linked to. Since the table and all of the other content is now nested within that li#exe02_117, and the li closing tag encompasses the table (and the figure, not shown in the code example), the entire answer will be available when the show-hide automation is performed.
