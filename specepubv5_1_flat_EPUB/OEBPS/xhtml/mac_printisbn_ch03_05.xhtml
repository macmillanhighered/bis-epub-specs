3.4. Revealed Extended Descriptions (Long Alt-Text)

3.4.1. Alt-text

In EPUB, alt-text is used to describe an image to users using a screen reader to access content. When alt-text is delivered to Macmillan, it is typically provided in a spreadsheet, so there is no distinction made between the two types of alt-text that we use.

Macmillan uses two types of alt-text:

  • Short alt-text, or simply alt-text, is an attribute applied to an image directly. It cannot be longer than 125 characters. It is important to note that short alt-text is a part of the image itself.
  • Long alt-text, or extended description, is a separate longer piece of content that is used to describe an image. Extended descriptions are not part of the images, but referred to by the image in order to describe the image.

3.4.1.1. Why two types of alt-text?

Short alt-text is used to describe simple images that can be described in under 125 characters.

Extended descriptions are used to describe much more complex images. These can include, but are not limited to, charts, graphs, maps, complex photos, or images with annotations. There are a separate set of guidelines for what images require extended descriptions.

Extended descriptions can include much richer semantic information than short alt-text. Multiple paragraphs, lists, and tables can be included in extended descriptions to provide even more context to the user.

3.4.1.2. How are they different?

Short alt-text is a part of the image.

Long alt-text is referenced by the image.

This is an image with short alt-text. The caption is completely separate from the image itself.

Below is the image as written in HTML (EPUB code).

            <img  src="../images/HUN_10344_01_P07.png"  alt="A photo shows the Minoan palace at  Knossos on Crete. It shows primarily stone  foundations and wall remnants, with a stone  staircase." />

The img element is used to represent the image in the markup.

Line 2 src (source) references the PNG image file.

Lines 3-6 contain the alt-text, which is what the screen reader reads. The short alt-text reads “A photo shows the Minoan palace at Knossos on Crete. It shows primarily stone foundations and wall remnants, with a stone staircase”. Since it is inside of the img element tag, it is part of the image, not merely referenced by the image.

Extended descriptions on the other hand refer the user to a separate piece of content which contains the long description of the img.

3.4.2. Changes to extended descriptions (long alt text)

Benetech, the organization that certifies the accessibility of our EPUBs, has recommended updating our method of marking up extended descriptions.

Currently, we have the extended descriptions on the same page, but essentially hidden out of the reading frame. This is no longer best practice for extended descriptions because this can interfere in how screen readers access the content and because it does not provide access to sighted users who may benefit from further description of an image.

3.4.2.1. New method

An information icon (which is in Macmillan’s Common Design Language) will appear after the image. The information icon will be linked to separate page with the extended description.

Each extended description will be on its own page with a link back to the image it describes. All of the extended descriptions will be collected under a top-level heading in the EPUB navigation and TOC.

The link around the information icon will have a title attribute for sighted users to mouseover. (The screenshot below shows the mouseover text in the black box beneath the icon). The screen reader will read out the same information as the mouseover text, which is

Follow link to extended description of

followed by the name of the figure.

The image inside of the link will have alt-text that matches the mouseover text for screen readers to access.

Following the link on the icon will take the user to the extended description at the end of the book.

The extended description has a heading describing which figure it is referring to. That title will match with the figure number and title, if present. At the end of the description there will be a link back to the icon with text indicating to the user where the link returns them to.

The extended description will also contain the original image (for the ease of use of the sighted reader), but without the alt-text (to avoid double reading from the screen reader) and not set as a figure (to avoid having the image appear twice in the figure navigation pane).

3.4.2.3. Icon indicating a book has extended descriptions

All users will have to understand the purpose of the icon, so an explanatory note will need to be included. Please refer to the page Accessibility page for EPUBs for more instruction.

3.4.3. Rules

For alt-text under 125 characters, please use guidelines for short alt-text:

  • Alt-text should be added to the alt attribute of the img element

For alt-text that is complex or is longer than 125 characters, please use guidelines for long alt-text / extended description:

3.4.3.1. Navigating Extended Descriptions

  • All of the extended descriptions should be gathered at the end of the book under the top-level heading “Extended Descriptions” as a-heads
  • Extended Descriptions page must be created at the top level heading. It must appear as a node in the navigation with each extended description underneath it.
  • Extended Descriptions page contains an unordered list with links to each extended description in the book.
  • Extended descriptions must appear at the very end of the backmatter, right before the back cover and inside back cover (if present).
  • The extended description must have a heading describing which figure it is referring to. That heading will begin with “Extended Description for” followed by the name of the figure. The name of the figure should be the Figure type (figure, map, chart, etc) followed by an identifier (1, 1.1, 1A, etc) and the title of the figure in the caption (for example, Greece and the Aegean Sea, 1500 B.C.E.), if some or all of these elements are present. For example, if just the title is present, as in the below image, use the title: “Extended Description for The Domestication of Animals”.
    • Only if none of these elements are present may the name of the figure may be replaced with the short alt text.
  • The extended descriptions section must be accessible in the navigation of the EPUB

3.4.3.2. Tagging Figures

  • The figure should be followed by the “i” information icon from the Macmillan Common Design Language
  • That icon should link to an extended description on a separate XHTML page

3.4.3.3. Treatment of Extended Descriptions

  • Extended description page content must not be hidden and must use current Schema
  • Extended description should use semantic elements when necessary (for example, if list or tabular content is needed to describe the image)
  • Extended descriptions must include links back to the image it is describing
  • Whenever extended descriptions are used in an EPUB, informational text (provided by Macmillan) must be included in the front matter of the book (indicated by Macmillan in conditional text) to describe the purpose of the extended description icon (Refer to Accessibility page for EPUBs)
